## 미결 사항

### Q1. 로그 파일 저장 기능 포함 여부
- 현재 계획: 클립보드 복사만 제공
- 옵션 A: 포함 안 함 (심플 유지)
- 옵션 B: 전송된 로그를 .txt 파일로 누적 저장 (감사 추적용)
- **결정 필요**

### Q2. 전송 간격 최소값 제한
- 현재 계획: 50ms ~ 5000ms 범위 조정 가능
- 50ms 미만 설정 허용 여부 확인 필요 (UDP burst 위험)
- **결정 필요**

### Q3. Person 픽스처 편집 가능 여부
- 현재 계획: 코드 내 하드코딩 픽스처
- 옵션 A: 코드 고정 (심플)
- 옵션 B: UI에서 Person 값 편집 가능
- 옵션 C: JSON/YAML 외부 파일로 관리
- **결정 필요**

### Q4. 다중 전송 대상 지원 여부
- 현재 계획: Host:Port 단일 대상
- 옵션: 복수 대상에 동시 전송 (BroadCast)
- **결정 필요**

### Q5. C3-NIT-0200 공통부 구조 확인
- 카드 전문은 뱅킹 공통부(101byte)와 구조가 다름
- `TLGRM_LEN(4) + TLGRM_TXT(3) + TID(20) + ...` 형태로 별도 공통부 적용
- 뱅킹의 `LogCommonBody`와 분리된 `CardCommonBody`로 확정 처리 필요
- **M1 시작 전 설계 확정 필요**
